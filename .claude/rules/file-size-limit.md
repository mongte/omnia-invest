# 파일 크기 제한

에이전트가 생성하는 파일이 지나치게 커지면 가독성과 유지보수성이 저하됩니다.

## 기준

- **300줄 이하**: 권장 (대부분의 파일이 이 범위 내에 있어야 함)
- **300~500줄**: 경고 — 분해 검토 필요
- **500줄 초과**: 금지 — 즉시 분해

## 분해 전략

컴포넌트가 500줄을 넘으면:
1. UI 관심사(마크업)와 로직(훅)을 분리
2. 반복되는 UI 블록은 하위 컴포넌트로 추출
3. 상태 로직은 `use*.ts` 커스텀 훅으로 추출
4. 타입 정의는 `types.ts`로 분리

서비스/유틸이 500줄을 넘으면:
1. 단일 책임 원칙(SRP)으로 함수 그룹을 파일로 분리
2. 관련 함수들을 같은 파일에 유지 (응집도 우선)

## 검사 방법

```bash
bash scripts/check-file-size.sh
```
