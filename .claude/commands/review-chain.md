현재 브랜치의 전체 리뷰 체인을 실행합니다.
추가 컨텍스트: $ARGUMENTS

## 전제 조건 확인

```bash
git diff main --name-only
```
변경된 파일이 없으면: "변경 사항이 없습니다" 출력 후 종료.

## 리뷰 순서

### 1단계: 코드 리뷰 (`/review`)
- diff 분석 (아키텍처, 타입 안전성, FSD 규칙 준수)
- 버그 가능성, 보안 이슈 검토

### 2단계: 아키텍처 검증
```bash
bash scripts/check-fsd-deps.sh
```
위반이 있으면 즉시 보고 (리뷰 계속)

### 3단계: 시각적 일관성 검토 (`/design-review`)
- 대상: http://localhost:3000 (dev 서버 실행 중일 때만)
- 스타일 가이드 준수, UI 일관성

### 4단계: 기능 검증 (`/qa-only`)
- 대상: http://localhost:3000 (dev 서버 실행 중일 때만)
- 변경된 기능의 동작 확인

## 종합 판정

**APPROVE 조건** (모두 충족):
- 코드 리뷰: 치명적 이슈 없음
- FSD 위반: 0건
- 시각적 이슈: 없음 또는 경미한 수준
- 기능 QA: 핵심 케이스 통과

**REQUEST_CHANGES 조건** (하나라도 해당):
- 타입 에러 발생
- FSD 레이어 위반
- 핵심 기능 QA 실패
- 보안/접근성 심각 이슈

판정 결과와 각 단계별 피드백을 종합하여 출력합니다.
